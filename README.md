# s4 — 2F→1F 階段ワープ（saya モデル用・インライン版）

S-quattro の **既存タブにコードを貼り付けるだけ** で動く 2F→1F 階段ワープです。  
**外部 Python ファイル（.py）の配置は不要** です。

## レイヤー構成

| レイヤー名 | 階 | 役割 |
|------------|-----|------|
| `部屋` | 両階共通 | 人流の始点・終点 |
| `階段出口_2F` | 2F | 2F 階段ゴール。ここで 1F へワープ |
| `階段入口_1F` | 1F | 2F から降りてきたエージェントの出現位置 |
| `階段` | 1F | 1F 階段ゴール。ワープ後の次の目的地 |

## 手順（2か所だけ編集）

### 事前準備：地図エディタ

階段が3か所ある場合、それぞれについて次の **3つの領域** を作ります。

1. `階段出口_2F` … 2F・階段の奥のノード **1つ** を囲む
2. `階段入口_1F` … 1F・上からの合流点のノード **1つ** を囲む
3. `階段` … 1F・階段ゴールのノード **1つ** を囲む

**推奨**: 対応する3領域に同じ属性 `stair_id` を付ける（例: `BR`, `TR`, `CL`）。  
未設定の場合はレイヤー内の領域を座標順に自動ペアリングします。

---

### 手順 1：ミクロ人流地図「環境の初期化後の処理」

1. S-quattro でモデル `saya` を開く
2. **ミクロ人流地図** 部品を選択
3. **環境の初期化後の処理** タブを開く
4. 既存コードがあれば **そのまま残し**、なければ空でもよい
5. `snippets/map_init_after.py` の **全文** を貼り付ける

**注意**

- `def initAfter(...):` は **絶対に書かない**（タブの内容がそのまま実行される）
- `from stair_warp import ...` などの import は **不要**

**確認**: シミュレーション開始後、ログに次が出ること

```
=== [MAP] STAIR_WARPS_2F_1F ===
BR up 39 (...) entrance 51 (...) 1F_goal 60 (...)
...
=== [MAP] STAIR_WARPS_2F_1F END (count=3) ===
```

`count=0` や `WARN` が多い場合はレイヤー名・領域内のノードを確認してください。

### 手順 2：ミクロ人流エージェント「エージェントのステップ処理」

1. **ミクロ人流エージェント** 部品を選択
2. **エージェントのステップ処理** タブを開く
3. 既存コードの **一番下（末尾）** に `snippets/agent_step.py` の内容を追記

**注意**

- 既存のステップ処理を **削除しない**
- import 文は **追加しない**

### 手順 3：人流設定

| 対象 | 始点 | 終点 |
|------|------|------|
| 2F エージェント | `部屋`（2F 側の領域） | `階段出口_2F` |
| 1F エージェント | `部屋`（1F 側の領域） | `階段` |

- `proc0`（ランダム経路の無限生成）は **使わない** 構成を推奨
- テスト時は生成 **1人** から始めると確認しやすい

### 手順 4：一般／高齢者の速度差と色分け（推奨：パラメータ方式）

論文と同様に **一般** と **高齢者** を分け、高齢者比率をパラメータで切り替えられます。

```python
elderlyRate = self.env.simulator.param.elderlyRate
```

人流機能（`proc2`）・エージェント集合・エージェント生成後の処理の **すべて** で同じ `param` を参照します。

#### 4-1. シミュレーションパラメータを追加

S4 の **パラメータ編集** → **パラメータタブ** で次を定義する。

| パラメータ名 | 型 | 例 | 意味 |
|--------------|-----|-----|------|
| `elderlyRate` | 実数 | `0.3` | 高齢者の割合（0〜1） |
| `v0General` | 実数 | `1.2` | 一般の最適速度 [m/s] |
| `v1General` | 実数 | `1.5` | 一般の最高速度 [m/s] |
| `v0Elderly` | 実数 | `0.8` | 高齢者の最適速度 [m/s] |
| `v1Elderly` | 実数 | `1.0` | 高齢者の最高速度 [m/s] |

- 一括実行・感度分析で `elderlyRate` だけ動かすと、論文のシナリオ比較がしやすい
- パラメータが無い場合はスニペット内のデフォルト値（`elderlyRate=0.3` など）が使われる

#### 4-2. コードを貼る

| ファイル | S4 上の貼り付け先 |
|----------|-------------------|
| `snippets/agent_init_after.py` | **エージェントの初期化処理**（速度・色・elderlyRate） |
| `snippets/agentset_init_after.py` | **エージェント集合の初期化処理**（任意・ログ確認用） |

**「エージェントの初期化処理」の開き方**

1. モデル編集画面で **ミクロ人流エージェント** 部品をクリック
2. 右側（または下部）のプロパティ一覧をスクロール
3. **「エージェントの初期化処理」** の行を探す（「ステップ処理」の上あたり）
4. 右の **「編集」** ボタンを押す → コード編集タブが開く
5. そこに `agent_init_after.py` の全文を貼る

※「生成後の処理」というタブは **ありません**。正しくは **「初期化処理」** です。

**色分け（重要）**

| 区分 | 色 | マーカー |
|------|-----|----------|
| 一般 | 青 `#2980B9` | 丸 `o` |
| 高齢者 | オレンジ `#E67E22` | 三角 `^` |

`screenColor` だけでは **地図上の色は変わりません**。次も設定する。

| ファイル | 貼り付け先 |
|----------|------------|
| `snippets/map_draw_agents.py` | **ミクロ人流地図** > **環境のエージェントの描画処理** |
| `snippets/agent_step_color.py` | **エージェントのステップ処理** の末尾（任意・保険） |

#### 4-3. 人流 proc2 について

`generateAgentsAndSetDest()` は **`v0` 引数を受け取れない**（TypeError になる）。  
速度・高齢者比率は **`agent_init_after.py`（エージェントの初期化処理）** で設定する。

`proc2` の `innerproc` は次の1行のままでよい。

```python
generateAgentsAndSetDest(_rp, lns_[1:])
```

位置ばらつきを使う場合は `peopleflow_elderly_innerproc.py` を参照（`v0=` は付けない）。

#### 4-4. 代替：人流行を 2 本に分ける場合

`proc2` を触りたくない場合は、人流 UI でタイプ 0（一般）とタイプ 1（高齢者）の  
人流行を別々に作り、人数で比率を合わせてもよい。色分けは `agent_init_after.py` だけで動く。

#### 4-5. 確認

1. ログに `elderlyRate = 0.3` などが出る（`agentset_init_after.py` 使用時）
2. 画面で青（一般）とオレンジ（高齢者）が混在する
3. `elderlyRate` を `0.0` / `1.0` に変えて再実行し、比率が変わること

## 動作の流れ

```
2F: 部屋 → 階段出口_2F（領域に入る）
      ↓ setStaying(p=階段入口_1Fの座標) でワープ + 約5秒滞留
1F: 階段入口_1Fに出現 → 階段（1Fゴール）へ setDestination
```

- 階段降り時間は地図 initAfter 内の `_DEFAULT_STAIR_TIME = 5.0` で変更
- 各エージェントは `_stair_warped_2f_1f` フラグで二重ワープを防止

---

## トラブルシュート

| 症状 | 対処 |
|------|------|
| ログに `STAIR_WARPS` が出ない | 地図 initAfter が実行されているか、エージェント集合があるか確認 |
| `count=0` | レイヤー名の完全一致、各領域に経路地点が1つ入っているか確認 |
| `AttributeError: 'env'` | `self.env` ではなく **`self.agentset.env`** を使う |
| ワープしない | エージェント step に追記したか、2F の `階段出口_2F` に入っているか確認 |
| ワープ先がずれる | ログの `entrance` 座標と地図の `階段入口_1F` を照合 |
| ワープ後動かない | `1F_goal` がログに出ているか。`階段` レイヤーの領域・ノードを確認 |
| 色分けされない | **エージェントの初期化処理** に `agent_init_after.py` を貼ったか確認 |
| 速度差が出ない | **エージェントタイプ** の `v0` が一般／高齢者で異なるか確認 |
